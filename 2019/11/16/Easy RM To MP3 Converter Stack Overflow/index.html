0x02 详细调试步骤

下载 Easy RM To MP3 安装包后进行直接进行安装

Exploit-DB 上公开的 exp:

从作者的代码注释中可以了解到：header 和 junk 用于填充ESP 和 EBP 之间的内存空间，\xDB\x70\xBB\x01可以理解为 “jmp esp ” 指令地址，之后是220个 nop 指令和弹出计算器的 shellcode 指令。

根据作者公开的 exp 改写成 Python 脚本：

重新生成一个新的 evil1.m3u 文件

加载 evil1.m3u 文件，程序弹出对话框提示错误，说明该 exp 在该环境中无法使用。

要想在该环境中成功执行 shellcode，需要几个要点：

  1. 定位ret eip 地址
  2. 找到稳定的 jmp esp 地址
  3. shellcode 能正确地址

定位ret eip 地址，可以使用 windbg 的 mona 插件。

. load pykd.pyd

!py mona patten_create 30000

用生成HEX的 30000 字节为 junk 变量

重新生成 evil2.m3u，并用 windbg 附加到进程中，查看 ret地址

先运行 Easy RM To MP3 程序，然后附加到进程中。

在命令框中输入 g 运行程序程序，之后加载 evil2.m3u 文件，发现 ret 的偏移地址为 “\x48\x6d\x33\x48”

使用 mona 提供的 pattern_offset 插件，可以得到偏移地址为 5829

加成 20280 的偏移可以得到最终的偏移为 26109 (junk 偏移为 26105，其中 4 位留给 ebp）

重新生成 evil3.m3u 文件，按照布局，ret 的地址应该为0x43434343:

此时，esp 的地址为 0x000ff660

使用 mona 插件的 jmp -r esp -m kernel32.dll 查找 “jmp esp ”指令

选择 0x7576f7f7 作为 jmp esp 指令，用 0x7576f7f7覆盖 ret addr，shellcode 暂设为 \xcc 指令

生成 evil4.m3u 文件再次测试，程序在 int 3 指令处暂停
