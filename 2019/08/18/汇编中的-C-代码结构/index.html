0x07 函数调用约定

  1. cdecl 约定中，参数是从右到左被压入栈，当函数完成时由调用者清理栈，并且返回保存在 EAX 中

cdecl 函数调用

push c

push b

push a

call test

add esp, 2

mov ret, eax

  1. stdcall约定除了被调用者在函数完成时清理栈之外，其他和 cdecl 类似

srdcall 是 Windows API 的标准调用约定。任何调用这些 API 都不需要清理栈，因为清理栈的责任是有实现 API 函数代码的 DLL 程序所承担。

  1. fastcalt 约定中，前一些参数（典型的是前两个）被传到寄存器中，备用的寄存器是 EDX 和 ECX。
